📡 ML Canvas · Predição de Churn

Operadora Telecom · Modelo MLP (PyTorch) + Baselines · MLflow · API REST

👥 1. Stakeholders
  • Diretoria Executiva – redução do churn rate, ROI de IA
  • Diretor de Marketing – campanhas de retenção segmentadas
  • Diretor de CRM – integração do score de risco
  • Equipe de Produto – ajuste de ofertas e serviços
  • Time de Dados / MLOps – entrega da API, MLflow, monitoramento
  • Time Comercial – priorização de contatos com alto risco
🎯 2. Objetivos de Negócio
  • Reduzir churn rate em 15% nos primeiros 6 meses
  • Aumentar eficiência de campanhas: focar nos 20% clientes maior risco
  • Antecipar ações de retenção com 15 dias de antecedência
  • Reduzir CAC (custo de aquisição) através da retenção preventiva
  • Melhorar experiência e NPS com ações proativas
📊 3. Métricas de Negócio
  • Churn Rate – redução de 5 p.p. em 6 meses
  • Receita retida – R$ X milhões evitados
  • ROI da campanha → > 2.0 (receita retida / custo ação)
  • NPS – manter ou aumentar com ações de retenção
  • Taxa de conversão de ofertas para clientes com alto risco
⚙️ 4. SLOs (Service Level Objectives)
✅ Acurácia ≥ 85% 🎯 Recall (churn) ≥ 80% 📏 Precisão ≥ 70% 📈 AUC-ROC ≥ 0,90 ⚡ Inferência API ≤ 200ms (p95) 🛡️ Disponibilidade API ≥ 99,5% 🔄 Retreinamento mensal 📦 F1-score (classe churn) ≥ 0,75

Justificativa: recall elevado garante capturar a maioria dos cancelamentos; baixa latência permite integração com CRM e call center em tempo real.

🗃️ 5. Dados (Telco Customer Churn)
  • Fontes: CRM, faturamento, logs de uso, SAC, sistema de pagamentos
  • Volume total: 7.043 clientes (registros históricos)
  • Granularidade: cliente + atributos contratuais e demográficos
  • Janela de predição: churn nos próximos 30 dias
  • Target: Churn (Yes/No) → classe desbalanceada: ~26,5% churn
  • Tratamento: remoção de customerID, conversão de TotalCharges para float

🧠 7. Modelo(s)

🎯 Modelo principal: MLP (Rede Neural) com PyTorch

  • Arquitetura: Input → 2-3 camadas ocultas (Dropout + BatchNorm) → Sigmoid
  • Otimizador: Adam, função de perda: BCEWithLogitsLoss (com pesos para desbalanceamento)

📊 Baselines (Scikit-Learn):

  • Logistic Regression (class_weight='balanced')
  • DummyClassifiert
  • Comparação via Recall, AUC-ROC, F1-score

O modelo final será selecionado com melhor recall ≥80% e AUC-ROC ≥0.90.

📈 8. Rastreamento e Experimentação
  • MLflow Tracking – registro de hiperparâmetros, métricas (acurácia, recall, AUC-ROC), artefatos (modelos, curvas ROC, matriz confusão)
  • Comparação direta entre experimentos PyTorch vs Scikit-Learn
  • Versionamento de dados: DVC (opcional) para pipeline reprodutível
  • Log automático de métricas durante treino e validação
  • Registro do modelo campeão no MLflow Model Registry
🚀 9. Entrega / API
  • Framework: FastAPI
  • Endpoint: POST /predict – recebe JSON com features (após pré-processamento)
  • Resposta: {"churn_probability": 0.87, "high_risk": true, "risk_level": "alto"}
  • Documentação interativa
  • Latência garantida dentro do SLO (≤200ms p95)
🔍 10. Monitoramento e Governança
  • Data Drift: Evidently AI / Monitoramento de distribuição das features
  • Concept Drift: Avaliação contínua de métricas (recall, AUC) ao longo do tempo
  • Logs de predição: Armazenamento de inputs e scores para auditoria e rastreabilidade
  • Feedback Loop: Registro de cancelamentos reais (ground truth) para retreinamento programado
  • Versionamento de modelos: MLflow + Git, política de aprovação para produção
  • Alertas: Queda de recall abaixo de 75% ou aumento de latência acionam alertas automáticos
📌 11. Plano de Implementação (Roadmap)
  1. Exploração e preparação dos dados
  2. Feature engineering, encoding, normalização e split treino/teste
  3. Treinamento de baselines com Scikit-Learn + registro MLflow
  4. Construção e treino da MLP com PyTorch (otimização de hiperparâmetros)
  5. Comparação de modelos e seleção (Recall + AUC-ROC + trade-off negócio)
  6. Empacotamento do modelo campeão (formato TorchScript ou pickle + pré-processamento)
  7. Desenvolvimento da API (FastAPI, validação de entrada, documentação)
  8. Testes de carga e validação de SLOs (latência, throughput)
  9. Implantação em cloud/on-prem + pipeline CI/CD
  10. Monitoramento contínuo, retreinamento mensal e atualização do ML Canvas com métricas reais
